## 1. 产品概述

### 1.1 背景与目标
用户需要一款借助大模型创造力，快速生成个性化表情包的工具。用户只需输入简单关键词，系统即可自动扩写提示词、生成主题角色、**创建表情包系列**、批量生成表情描述，并最终生成静态表情图或动态 GIF 表情包。支持用户上传自有图片作为角色，支持对生成结果进行保存、管理和一键同步到微信表情开放平台、抖音表情包市场。

**新增概念：表情包系列**  
每次从选择生成类型（静态/动态）、输入系列关键词开始，到生成一批表情描述、批量产出表情包，构成一个“表情包系列”。系列会保存用户输入的系列关键词、生成类型、风格、生成的表情描述列表等关键信息，方便用户后期查看、重新生成或整体同步。

### 1.2 技术范围界定
- **动态表情实现方式**：调用图生视频模型生成 5 秒短视频 → 后端转 GIF。
- **角色一致性**：暂时由底层图像生成模型保证，不单独做微调或 LoRA。
- **内容审核**：依赖大模型服务商自带的安全过滤，系统在前置提交时增加第三方内容审核 API。
- **后期编辑**：V1.0 不提供编辑功能。

### 2.1 用户角色

- 用户**保存率 ≥ 30%**（生成后点击保存/同步的比例）。
- 角色一致性满意度（后续通过问卷收集）≥ 70%。

## 4. 关键字详细说明

用户在整个创作流程中会输入三类关键字，每种关键字的作用和存储位置如下：

| 关键字类型       | 出现阶段      | 用途                                                         | 示例                           | 存储位置                                                     |
| ---------------- | ------------- | ------------------------------------------------------------ | ------------------------------ | ------------------------------------------------------------ |
| **角色关键词**   | M1 生成角色IP | 用户输入的简短描述，系统将其扩写为详细的文生图提示词，用于生成角色图片。 | “一只戴眼镜的橘猫，很严肃”     | `cloud_emoji_ip_info.source_keyword`                         |
| **系列关键词**   | M2 创建系列   | 用户输入的简短核心词汇，大模型基于此批量生成一组结构化的表情描述。支持多个关键词用逗号分隔。 | “打工日常, 熊猫卖萌, 春节斗图” | `cloud_emoji_group.series_keywords`                          |
| **表情描述修改** | M2 编辑描述   | 用户对单个表情描述的四个字段（名称、情境、动作、元素）进行手动修改。 | 将“裂开”改为“崩溃大哭”         | 更新`cloud_emoji_group.description_list` 中的对应 JSON，以及后续生成的表情包表中的独立字段 |

**系列关键词的详细说明**  
- 用户输入系列关键词后，系统调用大模型，根据关键词生成 N 个（3/6/9/12）表情描述，每个描述包含`name`、`context`、`action_detail`、`expression_elements`。
- 例如：系列关键词 = “打工日常” → 可能生成表情描述：“周一早晨”、“开会摸鱼”、“加班崩溃”等。
- 系列关键词会保存在表情包系列表（`cloud_emoji_group`）的`series_keywords`字段中，方便用户日后回顾或基于相同关键词重新生成。

## 5. 功能模块详细需求

### M1 角色管理

#### 5.1 AI 生成角色
- **用户输入**：角色关键词（≤50 字符）
- **处理流程**：
  1. 调用大模型（如豆包）将角色关键词扩写为英文图像提示词，**展示给用户并允许修改**。
  2. 用户选择生成数量（1~4 张，默认 2 张）、图片比例（1:1 / 3:4 / 9:16，默认 1:1）。
  3. 调用文生图模型（如 sendance）生成候选图。
- **输出**：候选图网格展示，用户选择一张进入保存。

#### 5.2 本地上传角色
- **限制**：jpg/png/webp，≤10MB。
- **处理**：不做任何去背景、裁剪、风格转换。上传后直接作为角色图。
- **流程**：选择文件 → 预览 → 确认使用 → 进入保存。

#### 5.3 保存角色
- **保存字段**（对应 `cloud_emoji_ip_info`）：
  - `ip_id`（自动生成）
  - `user_id`（当前登录用户）
  - `name`（用户自定义，默认`我的角色_时间戳`）
  - `image_url`（角色图片 URL）
  - `source_type`（`ai_generated` / `user_upload`）
  - `source_keyword`（AI 生成时记录用户输入的角色关键词）
  - `description`（可选，用户补充的角色描述）
  - `create_time`、`update_time`、`status`
- **存储限制**：暂不限制每个用户保存的角色数量。
- **交互**：保存后自动跳转至创作页，并选中该角色。

#### 5.4 我的角色列表
- **展示**：网格布局（缩略图 + 角色名称）
- **操作**：点击选中、删除、重命名、设为默认角色
- **删除二次确认**：“基于该角色生成的表情包系列与表情包仍保留，但无法继续用此角色生成新系列。确定删除？”

#### 5.5 选择角色进行创作
- 用户必须至少有一个角色，否则引导创建。
- 创作过程中不允许修改角色图（需返回列表重新选择）。

---

### M2 表情包系列创建与描述生成

#### 5.6 系列创建（新增步骤）
- **前置条件**：已选定一个角色。
- **第一步：选择生成类型**  
  用户在角色选择页面**必须选择本次生成的类型**：静态表情 / 动态表情。选定后不可中途切换（除非返回角色页重新开始）。
- **第二步：输入系列关键词**  
  用户输入系列关键词（≤100 字符），支持多个关键词用英文逗号分隔。系统将基于这些关键词生成表情描述。
- **第三步：选择描述数量**  
  用户可选择生成 3/6/9/12 个表情描述，**默认 9 个**。
- **第四步：创建系列记录**  
  系统在后台创建一条 `cloud_emoji_group` 记录：
  - `emoji_group_id`（自动生成）
  - `ip_id`（当前选中的角色 ID）
  - `series_keywords`（用户输入的系列关键词）
  - `emoji_type`（静态 / 动态）
  - `emoji_style`（可选，用户选择风格，如“可爱”“写实”，默认为空）
  - `description_list`（暂时为空，待生成后填充）
  - `create_time` 等

#### 5.7 生成表情描述列表
- **调用大模型**：输入系列关键词 + 要求生成指定数量的结构化 JSON 数组，每个元素包含：
  ```json
  {
    "name": "表情名称",
    "context": "情境文字",
    "action_detail": "动作细节",
    "expression_elements": "表情元素"
  }
  ```
- **展示**：卡片列表，每个卡片显示四个字段。支持**编辑、删除、新增、拖拽排序**。
- **编辑时**：用户可修改任意字段，系统实时更新前端展示，并在用户点击“保存描述”时将最终列表提交到后端，更新 `cloud_emoji_group.description_list` 字段（JSON 格式）。
- **草稿保存**：描述列表（含顺序和手动修改）自动保存在本地 localStorage，并与当前 `emoji_group_id` 绑定，防止意外丢失。

#### 5.8 切换角色时的状态处理
- 若用户未保存当前系列（即未点击“保存描述”），切换角色时按原 M2 规则二次确认。
- 若用户已保存系列，切换角色时提示“切换角色将创建新系列，是否放弃当前系列？” 放弃则清空前系列，保留则保存当前系列到历史。

#### 5.9 进入生成阶段
- 描述列表至少有一条（否则“生成表情包”按钮禁用）。
- 用户需**勾选**要生成的表情描述（默认全选）。
- 点击“生成表情包”后，根据系列的 `emoji_type` 调用对应的生成模块（M3 或 M4），并传入系列 ID。

### M3 静态表情生成

#### 5.10 批量生成规则
- 用户勾选 N 个表情描述（1 ≤ N ≤ 12），系统依次（或并发）调用图生图接口，每个描述生成一个静态表情包。
- **前端展示**：批量生成进度条（例如“正在生成 3/9”），每个生成完成后实时显示缩略图预览。

#### 5.11 静态表情生成参数
- **模型**：图生图模型（如 SD + ControlNet / IP-Adapter）。
- **输入**：角色图（从 `cloud_emoji_ip_info` 获取） + 表情描述（将四个字段拼接为完整提示词）。
- **输出**：PNG 图片，尺寸 512x512，透明背景。
- **生成成功后**：创建 `cloud_emoji_info` 记录，**将描述的四个字段分别存入独立列**：
  - `emoji_id`（自动生成）
  - `user_id`、`ip_id`、`emoji_group_id`
  - `name`（从表情描述的 `name` 字段复制）
  - `context`（情境文字）
  - `action_detail`（动作细节）
  - `expression_elements`（表情元素）
  - `type`（静态）
  - `file_url`、`file_size`
  - `sync_status`（默认为 `none`）

> **重要**：不再使用 JSON 字段，四个描述字段直接作为表列，便于查询和展示。

#### 5.12 失败处理
- **部分成功**：批量生成中某个描述生成失败，其余成功生成的表情包正常创建记录并展示。失败条目显示“生成失败，点击重试”按钮，允许单条重试。
- **全部失败**：提示“生成失败，请检查网络或稍后重试”。

#### 5.13 生成后预览
- 所有生成结果以**网格形式**展示，与用户勾选的顺序一致（按系列中的 `description_list` 排序）。
- 每个表情包下方显示：
  - 表情名称（来自 `name` 字段）
  - 操作按钮：**保存至云端**（已自动保存，无需再点，但可提供“重新生成”）、**同步到平台**

> 注意：表情包生成后会自动保存到 `cloud_emoji_info`，用户无需手动保存。但可在预览页进行“重新生成”或“删除”。

---

### M4 动态表情生成

#### 5.14 动态表情生成参数
- **模型**：图生视频模型（如 Stable Video Diffusion、Pika Labs API）。
- **输入**：角色图 + 表情描述（同静态）。
- **输出**：5 秒视频，10fps，分辨率 240x240 或 320x320（由用户在生成类型选择时决定，默认 320x320）。
- **转 GIF**：后端或前端转为 GIF，帧率保持 10fps，**循环播放**（loop=0）。
- **GIF 优化**：256 色，文件大小 ≤ 5MB。
- **生成后记录**：同样创建 `cloud_emoji_info`，`type` 为 `dynamic`，`file_url` 存储 GIF 文件地址，同时写入四个描述字段。

#### 5.15 批量生成与预览
- 规则同 M3 5.10~5.13。
- 预览时展示 GIF 动图，鼠标悬停自动播放。

### M6 保存、资产管理及平台同步

#### 5.16 角色 IP 查询与管理
- **入口**：个人中心 → 我的角色 IP
- **列表展示**：网格布局，显示角色图片、名称、基于该角色的表情包系列数量（统计 `cloud_emoji_group`）。
- **查询功能**：
  - 按角色名称搜索
  - 按来源类型筛选（AI生成 / 本地上传）
  - 按创建时间排序
- **角色详情页**：展示角色信息 + 基于该角色的所有表情包系列列表（每个系列显示系列关键词、创建时间、表情包数量），点击系列可查看该系列下的所有表情包。
- **操作**：编辑角色名称、设置默认角色、删除角色（二次确认）。

#### 5.17 表情包系列查询与管理
- **入口**：个人中心 → 我的表情包系列
- **列表展示**：每个系列卡片显示：系列关键词、类型（静态/动态）、角色缩略图、表情包数量、创建时间。
- **查询功能**：
  - 按系列关键词搜索
  - 按类型筛选
  - 按角色筛选
- **点击系列**：进入系列详情页，展示该系列的所有表情包（网格），每个表情包卡片显示表情名称、情境、动作细节、表情元素（可折叠或小字显示）。支持批量操作（下载、删除、同步）。
- **系列操作**：删除整个系列（二次确认，同时删除系列下所有表情包资产）、重新生成系列（基于原角色和系列关键词，重新生成描述列表，覆盖原描述）。

#### 5.18 表情包资产查询与管理
- **入口**：个人中心 → 所有表情包（扁平列表）
- **列表展示**：网格布局，显示缩略图、名称、所属系列、同步状态。
- **查询功能**：
  - 按名称搜索
  - 按类型筛选
  - 按同步状态筛选
  - 按所属系列筛选
  - 按描述字段筛选（如情境文字包含“上班”）
- **批量操作**：勾选多个表情包，支持批量删除、批量同步到平台（统一选择微信或抖音）。
- **单个操作**：下载、删除、查看详情（展示四个描述字段）、重新生成（调用原系列对应的生成参数）、分享链接、同步。

#### 5.19 同步到微信表情开放平台

- **创作者账号要求**：用户需在微信表情开放平台完成创作者入驻。
- **格式规范**：
  - 数量：8~24 张（推荐 16 张）
  - 主图 240×240 PNG ≤500KB，透明背景
  - 封面 120×120，详情页横幅 750×400
  - AIGC 水印自动添加
- **同步流程**：
  1. 用户选择一个表情包系列（系列下必须至少有 8 个表情包）。
  2. 点击“同步到微信” → 微信授权登录。
  3. 填写系列元数据（名称、简介、标签等，系统自动从系列关键词和表情描述中提取默认值，简介可自动组合各表情的`context`字段）。
  4. 系统自动处理图片格式、打包 ZIP。
  5. 由于微信无公开批量 API，V1.0 采用**半自动方案**：提供下载链接，引导用户手动上传至微信表情助手小程序。用户上传后，在系统内点击“我已上传”，系统更新该系列所有表情包的 `sync_status` 为 `synced_to_wechat`，并记录同步时间。
  6. 后续版本若微信开放 API，则升级为全自动。

#### 5.20 同步到抖音表情包市场

- **创作者资质要求**：完成抖音实名认证，申请「im 代用素材上传」能力。
- **格式规范**：PNG/GIF/JPG/BMP，≤10MB，宽高 <1500px。
- **同步流程**：
  1. 用户选择一个或多个表情包（或整个系列）。
  2. 点击“同步到抖音” → 抖音 OAuth 授权。
  3. 选择内容标签（最多 2 个）。
  4. 系统自动添加 AIGC 水印和 EXIF 元数据。
  5. 调用抖音素材上传接口 `/enterprise/media/upload/` 批量提交。
  6. 同步成功后更新 `cloud_emoji_info.sync_status` 为 `synced_to_douyin`，记录 `sync_douyin_id`。
- **若同步失败**：记录失败原因，支持重试。

#### 5.21 合规性要求
- 内容原创性检测：版权关键词过滤（禁止迪士尼、漫威等 IP），前端提示风险。
- 内容安全：集成第三方内容审核 API（涉政、色情、暴力），生成后自动检测，违规图片拒绝保存并提示用户修改关键词。

## 7. 非功能性需求

### 7.1 性能要求
- 文生图（单张）平均时间 ≤ 15 秒。
- 图生视频（5秒）平均时间 ≤ 30 秒；转 GIF 额外 ≤ 5 秒。
- 页面首屏加载 ≤ 2 秒。

### 7.2 并发与资源控制
- 限制同一用户同时进行的生成任务数（最多 2 个），超出排队。
- 批量生成使用后端消息队列，前端轮询状态。

### 7.3 数据安全
- 用户角色图、表情包仅用户本人可见（默认私有）。
- 所有 API 调用走后端代理，不暴露密钥。

### 7.4 可用性
- 所有生成操作显示 loading 和预计剩余时间。
- 错误提示明确（如“生成失败：图片尺寸超出限制”）。

### 7.5 兼容性
- 支持 Chrome / Edge / Safari 最新版，移动端 H5 适配。

## 8. 异常处理汇总

| 异常场景               | 处理方式                                         |
| ---------------------- | ------------------------------------------------ |
| 大模型提示词扩写超时   | 重试 1 次，仍失败则提示用户手动输入              |
| 文生图/图生视频超时    | 提示失败，允许单条重试                           |
| 上传图片格式错误       | 前端校验，提示仅支持 jpg/png/webp                |
| 上传图片超过 10MB      | 前端拒绝并提示压缩                               |
| 生成 GIF 超过 10MB     | 提示降低分辨率或帧率，或自动降低质量             |
| 同步到微信时用户未入驻 | 引导用户前往微信表情助手小程序完成入驻           |
| 同步到抖音时授权失败   | 提示重新授权，并记录错误原因                     |
| 内容审核不通过         | 提示违规内容，拒绝保存/同步，并建议修改关键词    |
| 用户删除包含系列的角色 | 系列变为孤儿（ip_id 无效），前端显示“角色已删除” |

| 图生视频   | 以一张图片为参考，生成动态视频的技术                         |
